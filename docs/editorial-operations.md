# 편집·운영 기준

5ft magazine은 정기 발행일이나 응모 마감일을 약속하지 않는다.
대신 방문자가 다시 올 이유가 사라지지 않도록 아래 신호를 운영 기준으로 사용한다.

## 콘텐츠 우선순위

1. 필름 정보와 실제 촬영 결과
2. 필름·사진에 관한 Articles
3. 독자 사진 참여와 현재 호 주제
4. Magazine 소개와 구매
5. Market·Labs·Shop 보조 서비스

새 기능을 제안할 때는 위 다섯 항목 중 어느 축을 강화하는지 먼저 적는다.
어느 축에도 해당하지 않으면 기능보다 기사·사진 정리를 우선한다.

## 약속 없는 운영 리듬

- 새 글이 없어도 최근 독자 사진과 랜덤 추천 필름이 홈을 갱신한다.
- 45일 이상 새 글이 없으면 기존 글 재편집, 필름 묶음, 짧은 편집부 소식 중 하나를 검토한다.
- 현재 호 응모를 받을 때만 `current-theme.json`을 활성화한다.
- 유지할 수 없는 뉴스레터는 계속 숨기며, 실제 발송 체계가 준비된 뒤에만 다시 노출한다.
- 발행일·마감일은 확정할 수 있을 때만 화면에 표시한다.

`npm run content:health`는 최근 발행량, 주제 분포, 현재 호 활성 상태를 보여준다.
이 명령은 경고만 제공하며 발행을 강제하지 않는다.

## 정보 구조 측정

다음 익명 이벤트로 30일 단위 사용 흐름을 확인한다.

- `nav_clicked`: Articles·Films·Magazine·Market·More 실제 이용 비율
- `film_finder_searched`: PC 홈 필름 검색 사용량
- `film_finder_recommendation_clicked`: 추천 필름 진입량
- `shop_magazine_link_clicked`: Shop에서 Magazine으로 이동한 횟수

판단 기준은 절대 클릭 수보다 페이지뷰 대비 비율이다. More 안의 메뉴 사용이 지속적으로
높으면 상단 복귀를 검토하고, 낮다는 이유만으로 페이지를 삭제하지 않는다.

## 월 1회 점검

- 최근 글과 사진이 홈에서 정상 노출되는가
- Film Finder 검색 결과가 실제 필름을 찾는가
- 현재 호 문구와 응모 상태가 일치하는가
- Magazine의 미리보기·열람·구매 링크가 유효한가
- Shop의 책 상품에서 “이 호의 내용 보기”가 연결되는가
- Market 신고·개인정보 보호 정책이 정상인가
- `npm run qa`, `npm run db:audit`, `npm run content:health`가 정상인가
