개인정보처리방침

5ft.mag (운영자: 4rest) 는 회원의 개인정보를 최소한으로 수집·이용하며, 아래 방침에 따라 안전하게 보관·처리합니다.

최종 개정일 2026.05.15

1. 수집하는 개인정보 항목

운영자는 다음의 정보를 수집합니다.

  • Google OAuth 로그인 시 — 이메일 주소, Google 표시명, 프로필 사진 URL
  • 사진 투고 시 — 표시 이름 (선택), 인스타그램 ID (선택), 사진 파일, 카메라·필름 정보, 한 줄 메모, 게재 동의 여부
  • 매물 등록 시 — 매물 제목·가격·설명, 상품 사진, 거래 지역 (선택), 연락처 (회원 본인이 직접 입력)
  • 좋아요/스크랩 — 회원이 좋아요한 사진·필름·글의 식별자
  • 자동 수집 — 접속 시간, 방문 경로, 외부 유입 도메인, 브라우저 종류, 세션 식별자, 언어 설정, 시간대, 페이지 체류 시간, IP 주소(축약된 형태)는 보안·트래픽 분석 목적으로 Supabase 로그에 일시 저장

2. 개인정보 이용 목적

  1. 회원 인증·식별 (Google OAuth)
  2. 회원이 투고한 사진의 게재 및 종이 매거진 수록
  3. 매물 등록·열람·신고 처리
  4. 좋아요·스크랩 등 회원 본인용 컬렉션 관리
  5. 편집부의 콘텐츠 검토 및 부적절한 게시물 처리
  6. 법령 또는 약관 위반 행위 조사·제재
  7. 익명화·최소화된 접속 통계를 통한 콘텐츠 편집 및 서비스 개선

3. 개인정보 보관 기간

  • 회원 계정 — 회원 탈퇴 시까지. 회원이 본인 페이지(/me.html) 에서 사진·매물을 회수하면 즉시 삭제됩니다.
  • 승인된 사진 — 회원이 직접 삭제하기 전까지, 또는 종이 매거진 게재 후 매거진 발행 종료 시까지 보관.
  • 매물 — 회원이 직접 삭제하거나 운영자가 약관 위반으로 삭제할 때까지.
  • 신고 데이터 — 처리 완료 후 1년 보관 (분쟁 대응·재발 방지 목적).
  • 접속 로그·체류 시간 로그 — 기본 365일 이내 보관 후 운영자가 정기 삭제할 수 있습니다. 보안 조사나 장애 대응에 필요한 경우 관련 법령과 목적 범위 안에서 필요한 기간 동안 보관할 수 있습니다.
  • 법령 의무 보관 — 관계 법령에서 보관을 의무화한 경우 해당 기간(예: 전자상거래법상 표시·광고 기록 6개월).

4. 개인정보의 제3자 제공

운영자는 회원의 개인정보를 외부 제3자에게 제공하지 않습니다. 단, 다음의 경우 예외로 합니다.

  • 회원이 사전에 동의한 경우
  • 법령 또는 수사기관의 요구가 있는 경우 (영장·서면 요청 등 법적 근거가 있을 때에 한함)

5. 개인정보 처리 위탁

운영자는 서비스 운영에 필요한 일부 처리를 외부에 위탁합니다.

  • Supabase Inc. (미국) — 회원 인증·데이터베이스·파일 저장 (사진·매물 이미지)
  • Google LLC (미국) — OAuth 인증
  • Netlify Inc. (미국) — 사이트 호스팅 · CDN

위탁업체는 운영자와 체결한 약관·법령에 따라 개인정보를 안전하게 처리하며, 운영자는 위탁업체가 책임을 다하는지 정기적으로 점검합니다.

6. 회원의 권리

회원은 언제든지 다음의 권리를 행사할 수 있습니다.

  • 열람 — 회원 페이지(/me.html) 에서 본인이 투고한 사진·매물·좋아요·스크랩 데이터 확인
  • 수정 — 회원 페이지에서 사진 메타데이터·매물 정보 등 직접 수정
  • 삭제 — 회원 페이지에서 사진·매물·좋아요·스크랩 개별 삭제. 회원 탈퇴 요청은 4rest_design@naver.com 으로 보내주세요.
  • 처리 정지·반대 — 동일 이메일로 처리 정지 요청 가능

7. 개인정보 안전 조치

  • 모든 통신은 HTTPS 로 암호화됩니다.
  • Supabase 의 Row Level Security (RLS) 로 회원 데이터는 본인만 조회·수정 가능합니다.
  • 접속 통계 원본 데이터는 직접 공개하지 않으며, 편집부 권한이 있는 계정만 집계 화면을 열람할 수 있습니다.
  • 외부 유입 정보는 전체 URL이 아닌 도메인 단위로 최소화해 저장합니다.
  • 비밀번호 정보는 저장하지 않으며, 인증은 Google OAuth 토큰 기반입니다.
  • 편집부는 별도 권한이 부여된 계정만 접근 가능하며, 처리 활동은 로그로 남습니다.

8. 쿠키·로컬 저장소 사용

  • 로그인 상태 유지(localStorage), 테마 설정, 폼 임시 저장(sessionStorage), 방문 세션 구분(sessionStorage) 목적으로 브라우저 저장소를 사용합니다.
  • 광고·트래킹용 쿠키는 사용하지 않습니다.
  • 사용자는 브라우저 설정으로 언제든지 쿠키·저장소를 삭제할 수 있습니다.
